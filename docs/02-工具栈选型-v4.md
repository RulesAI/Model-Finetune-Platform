## 三、Phase 0 CPT 工具详解（v4 新增）

### 3.1 ms-swift 的 CPT 支持

ms-swift v4 官方描述："Use PEFT or Full-parameter to **CPT**/SFT/DPO/GRPO 600+ LLMs"

**已原生支持 CPT**，命令：

```bash
swift pt \
  --model_id_or_path <model> \
  --train_type full \
  --dataset <cpt_corpus> \
  --deepspeed zero3 \
  --learning_rate 2e-6 \
  --num_train_epochs 1
```

### 3.2 为什么 CPT 不能用 Unsloth

| 维度 | Unsloth | DeepSpeed ZeRO-3 |
|------|---------|------------------|
| 数据规模 | 小（< 1 亿 token）| 大（10 亿+ token）|
| GPU 数 | 单卡 | 多机多卡 |
| 模型分片 | 不支持 | 全套（参数 / 优化器 / 梯度分片）|
| 稳定性 | 多卡升级中 | 工业级稳定 |

CPT 阶段强制 DeepSpeed ZeRO-3，**禁用 Unsloth**（F1 校正落实）。

### 3.3 CPT 数据格式

```jsonl
{"text": "《黄帝内经·素问》... 全文 ..."}
{"text": "中药白术，性味苦甘温 ... "}
{"text": "期刊论文：基于网络药理学的 XX 研究 ... "}
```

纯 next-token prediction，不需要 instruction 格式。

### 3.4 HF Datasets streaming（v4 新增）

CPT 数据量 8-10 亿 token，普通模式会撑爆内存。必须 streaming：

```python
from datasets import load_dataset

dataset = load_dataset(
    "json",
    data_files="data/cpt/tcm/*.jsonl",
    streaming=True,  # 关键
    split="train"
)
```

```python
def check_backend(scene_config):
    tokens = scene_config['estimated_tokens']
    backend = scene_config['backend']

    if backend == 'unsloth' and tokens > 100_000_000:
        raise ValueError(
            f"Scene {scene_config['scene']}: {tokens:,} tokens > 1B threshold, "
            f"switch to deepspeed zero3"
        )
```

### 4.2 F2 — QualBench 前置 baseline


**判定规则**：

## 六、大规模数据处理工具（v4 新增）

### 6.1 问题

TCM 场景 10 万篇期刊论文 × 10 秒/篇 = **~12 天单机**，不可接受。

### 6.2 工具选项

| 选项 | 复杂度 | 适用规模 |
|------|-------|---------|
| **Python multiprocessing**（推荐起步）| 低 | 10 万-100 万文档 |
| Ray | 中 | 100 万-1000 万文档 |
| Spark | 高 | 1000 万+ |

**v4 推荐起步方案**：multiprocessing 包装 MarkItDown：

```python
# scripts/data_prep/parallel_markitdown.py
from multiprocessing import Pool
from markitdown import MarkItDown

def convert_one(path):
    md = MarkItDown()
    return md.convert(path).text_content

def parallel_convert(file_list, workers=24):
    with Pool(workers) as pool:
        results = pool.map(convert_one, file_list)
    return results
```

24 进程 × 0.4 秒/文档 = **~28 小时**完成 10 万篇（vs 单机 12 天，**10× 加速**）。

### 6.3 数据存储

| 数据规模 | 推荐存储 |
|---------|---------|
| < 1 GB | 单 jsonl |
| 1-100 GB | parquet 分片 |
| > 100 GB | webdataset / 对象存储 |

TCM 8 亿 token ≈ 4-8 GB → parquet 分片即可。
