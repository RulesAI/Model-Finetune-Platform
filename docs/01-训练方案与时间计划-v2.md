## 二、数据画像（通用 + HR 实例）

### 2.1 通用：场景数据画像模板

任何新场景接入本工具集前，必须填写：

| 维度 | 必填 |
|------|-----|
| 文档总数 / 总字节数 / 估算 token 数 | ✅ |
| 主题结构（子目录 / 标签 / 大小占比） | ✅ |
| 文档类型分布（md / pdf / docx / xlsx / 图片） | ✅ |
| 涉密等级 / 脱敏状态 | ✅ |
| 强引用网络密度（决定是否需要 GraphRAG） | ✅ |
| 多模态需求（图纸 / 工艺图 / 表格密度） | ✅ |

### 2.2 HR-Salary 场景实例

| 维度 | 数值 |
|------|------|
| 文件总数 | **158 个 markdown** |
| 总字节数 | 6,302,340 bytes ≈ **6.3 MB** |
| 估算 token 数 | 约 **140 万 token** |
| 主题结构 | 央企脱敏案例 / 政策汇编 / 书籍政策 |
| 引用网络密度 | **高**（39 号令 → 公司法 → 例外细则）→ 触发 LightRAG |
| 多模态需求 | 无（纯文本）|
| 涉密等级 | 高（已脱敏，需二次校验）|

## 三、关键判断（通用判断 + 场景实例）

### 3.1 通用判断 1：CPT 数据量阈值

| 阶段 | 行业典型数据量 | 阈值含义 |
|------|--------------|---------|
| **CPT（继续预训练）** | 10 亿 - 100 亿 token | 不到 1 亿 token 几乎无效 |
| **SFT（监督微调）** | 3k - 10 万条 instruction | 数据质量 > 数量 |
| **DPO（偏好对齐）** | 200 - 2000 偏好对 | 高质量专家审定 |

**HR-Salary 实例**：140 万 token 跑不了 CPT（差 1000 倍），直接 SFT 起步。

### 3.2 通用判断 2：RAG 必备性

**任何强引用 / 强事实精度的领域都必须配 RAG**：
- HR 政策（条款数字）→ 必需
- 制造 SOP（设备参数 / 操作步骤）→ 必需
- 供应链单据（数字精度 / 流程合规）→ 必需

微调能学风格 / 学结构 / 学推理，但事实精度必须 RAG 兜底。

### 3.3 通用判断 3：图增强 RAG 的触发条件

| 场景特征 | 是否触发 LightRAG |
|---------|------------------|
| 引用网络密集（如政策文档） | ✅ 必上 |
| 实体之间存在多跳关系（如供应链单据流） | ✅ 必上 |
| 平铺文档无强关联（如 FAQ）| ❌ 纯向量即可 |

**HR-Salary 实例**：政策文档有 5+ 层引用网络 → 触发 LightRAG。

## 四、模型版本推荐

### 4.1 推荐方案：**GLM-4-9B-Chat-1M**（主选）

| 维度 | GLM-4-9B-Chat-1M |
|------|------------------|
| 参数量 | 9B（90 亿） |
| 上下文 | **100 万 token** |
| 开源协议 | 智谱开源协议（商用需登记） |
| 硬件门槛 | LoRA：单卡 A100 80G；全参 SFT：4-8 卡 A100 80G |
| 适用场景 | HR / 政策密集 / 客户偏好国产 |

### 4.2 备选方案：**Qwen2.5-14B-Instruct**（Plan B，v2 新增）

| 维度 | Qwen2.5-14B-Instruct |
|------|---------------------|
| 参数量 | 14B |
| 上下文 | 128K（标准）/ 1M（Qwen2.5-Turbo 版） |
| 开源协议 | **Apache 2.0**（最宽松）|
| 适用场景 | 客户接受民企模型 / 需要更宽松协议 / 制造与供应链场景 |

### 4.3 升级路径

如需更强 → GLM-4-32B 或 Qwen2.5-32B（4× 显存）

## 五、性价比最高的任务清单（4 阶段流水线，v2 校正）

### Phase 1：数据工程（性价比 ★★★★★，无变化）

158 篇 md → 15k 条 instruction。流程见 v1 §四 Phase 1。

**多场景扩展**：每个场景在 `scenes/<scene>/instructions/rules.yaml` 配置自己的数据合成规则，复用通用 pipeline。

### Phase 2：SFT 监督微调（v2 大变更：**ms-swift 主**）

| 配置项 | 推荐值 | 变更 |
|--------|--------|------|
| **训练框架** | **ms-swift v4.0** | ⭐ v2 新（国企场景）|
| 训练备选 | LLaMA-Factory | v1 保留（对照基线）|
| 训练方法 | LoRA（r=16，alpha=32） | 不变 |
| 学习率 | 2e-4 | 不变 |
| Epoch | 2-3 | 不变 |
| batch size | 32（梯度累积）| 不变 |
| 最大序列长度 | 8192 | 不变 |
| 加速 backend | Unsloth（提速 60%） | 不变 |
| **配置文件路径** | `configs/ms_swift/scenes/<scene>/sft_glm4_9b.yaml` | ⭐ 路径调整 |

**ms-swift CLI 示例**：
```bash
swift sft \
  --model_id_or_path ZhipuAI/glm-4-9b-chat-1m \
  --sft_type lora \
  --dataset data/instructions/hr-salary/sft_v1.jsonl \
  --output_dir checkpoints/hr-salary/sft-v1 \
  --learning_rate 2e-4 \
  --num_train_epochs 3
```

### Phase 3：DPO 偏好对齐（v2 大变更：**ms-swift 主**）

```bash
swift rlhf \
  --rlhf_type dpo \
  --model_id_or_path checkpoints/hr-salary/sft-v1 \
  --dataset data/preferences/hr-salary/dpo_v1.jsonl \
  --output_dir checkpoints/hr-salary/dpo-v1
```

### Phase 4：RAG 知识库（v2 大变更：**双层 = 向量 + 图增强**）

| 层 | 工具 | 职责 |
|----|------|------|
| 文档解析 | RAGFlow DeepDoc | 多格式 → 结构化 markdown |
| 向量检索 | RAGFlow + BGE-M3 + ChromaDB | 语义相似召回 |
| **图增强**（v2 新增）| **LightRAG** | 提取实体 / 关系 / 引用网络，多跳推理 |
| Rerank | BGE-reranker-v2-m3 | 融合向量 + 图召回 |

**关键协同**：LightRAG 集成层**复用 GraphForge 团队经验**（详见主 CLAUDE.md「LightRAG 定制原则」）。

### 7.2 通用能力回归（必跑）

| benchmark | 抽样量 | 通过线 |
|-----------|-------|--------|
| C-Eval | 200 题 | 比 base 掉点 < 3% |

### 7.3 RAG 评估（v2 新增）

| 指标 | 工具 | 通过线 |

| Answer Relevancy | RAGAS | ≥ 0.80 |
| Context Precision | RAGAS | ≥ 0.75 |
| Context Recall | RAGAS | ≥ 0.80 |

### 7.4 人工评估（不变）

事实准确性 / 合规性 / 流畅度 / 有用性，1-5 分制，通过线 4.0+。

| 资源 | 推荐配置 | 预计用量 | 成本估算 |
|------|---------|---------|---------|

## 九、多场景演进路线（v2 新增）

### 9.1 第二个场景启动 checklist

新场景启动前必须完成：

- [ ] 业务方确认数据来源、规模、敏感性
- [ ] 填写"场景数据画像模板"（§2.1）
- [ ] 评估 RAG 是否需要图增强（§3.3）
- [ ] 评估是否需要多模态扩展（图纸 / 表格 / 图片）
- [ ] 选择基座（GLM / Qwen / 其他）
- [ ] 在 `scenes/<scene>/` 填充 prompts / instructions / eval_set
- [ ] 在 `configs/<tool>/scenes/<scene>/` 复制 hr-salary 模板并定制

### 9.2 三场景预期时间表

```
2026 Q2 (May-Jun)   |▓▓▓▓▓▓ HR-Salary 首发（6 周）
2026 Q3 (Jul-Sep)   |        ░░░▓▓▓ Manufacturing 启动（3-4 周）
2026 Q3-Q4 (Sep-Oct)|              ░░░░▓▓▓ Supply-Chain 启动（3-4 周）
```

第二、三个场景预期 **3-4 周交付**（比首发场景 6 周快约 50%），因为核心工具集已成熟。

### 9.3 工具复用率目标

| 阶段 | 工具复用率（场景间）| 场景特定代码比例 |
