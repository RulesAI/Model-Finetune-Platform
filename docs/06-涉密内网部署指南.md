## 三、硬件需求清单

### 3.1 训练机（Training Server）⭐ 核心投入

| 配置项 | 推荐配置 | 国产替代 | 关键参数 |
|--------|---------|---------|---------|
| GPU | **NVIDIA A100 80G × 8 卡**（或 H800 80G × 8 卡）| 华为昇腾 910B × 8（需架构改造，见 §5.3）| 显存 ≥ 80GB / 卡 |
| CPU | Intel Xeon Gold 32-64 核 | 华为鲲鹏 920 | 64-128 核 |
| 内存 | DDR5 ECC **512 GB** | - | ≥ 8× GPU 显存总量 |
| 存储 | NVMe SSD 4 TB + HDD 10 TB | - | 训练数据 + checkpoint 多版本 |
| 网络 | 节点内 NVLink 600 GB/s | - | 多卡训练必需 |
| 电源/散热 | 双路冗余 + 液冷可选 | - | 持续高负载 |

**为什么 8 卡 A100 而不是 4 卡**：
- LoRA SFT GLM-4-9B：4 卡够，但 F3 双基座 A/B 并行（GLM + Qwen2.5-14B）需 8 卡
- 14B 全参 SFT 兜底：4 卡紧
- Phase 0 CPT 工具兼容（HR 不做，但工具集需要兼容 TCM 等场景）

**采购 vs 租赁建议**：HR 5 周项目优先**租赁云 GPU**（涉密条件允许的话），多场景滚动建议采购。

### 3.2 推理机（Inference Server）

| 配置项 | 推荐配置 | 关键参数 |
|--------|---------|---------|
| GPU | **RTX 4090 24G × 1**（或 A10 24G × 1）| INT4 量化后 GLM-4-9B 仅 5 GB，单卡绰绰有余 |
| CPU | 16 核 | - |
| 内存 | 64 GB | - |
| 存储 | NVMe SSD 500 GB | - |
| 网络 | 10 Gbps 内网 | 对接业务系统 |

**HA 建议**：2 台推理机做主备（约 ¥20 万），避免单点失败。

### 3.3 业务机（Application Server）

| 配置项 | 推荐配置 |
|--------|---------|
| CPU | 2 路 Xeon 共 32 核 |
| 内存 | 256 GB |
| 存储 | 4 TB SSD |
| 用途 | FastAPI 后端 + Next.js 前端 + PostgreSQL + Argilla + Prometheus + Grafana |
| GPU | **无需** |

### 3.4 存储规划（HR 场景体积）

| 资源 | 大小 | 存放位置 |
|------|------|---------|
| 本项目代码 | ~10 MB | 业务机 |
| PyPI venv | ~5 GB | 训练机 + 业务机 |
| Docker images（5-6 个）| ~20 GB | Harbor Registry |
| 基座模型（GLM-4-9B + Qwen2.5-14B）| ~50 GB | 训练机 `/data/models/` |
| 训练数据中间产物 | ~1 GB | 训练机 |
| Checkpoints（5-10 个 LoRA adapter）| ~10 GB | NAS |
| 日志 + 评估报告 | ~5 GB | NAS |
| **总计** | **~90 GB** | - |

⚠️ 建议预留 **500 GB - 1 TB**（含多次重训 / A/B 对比 / 备份）。

## 五、合规与信创要求

### 5.1 等保级别评估

| 等保级别 | 客户类型 | 影响 |
|---------|---------|------|
| **二级等保** | 一般企业 | 基础安全配置 |
| **三级等保**（最常见）| 央企 / 国企 / 政府 | 完整审计 + 双因素认证 + 漏扫 |
| 四级等保 | 关键基础设施 | 物理隔离 + 国密强制 |

国企 HR 客户 **90% 三级等保**。

### 5.2 信创要求 4 等级

| 等级 | 要求 | 应对 | 工具集兼容性 |
|------|------|------|------------|
| **L0 无要求** | 用 NVIDIA + Ubuntu | 标准方案 | 100% |
| **L1 软件信创** | 麒麟/统信 OS + 达梦/OceanBase DB | NVIDIA 仍可 | 95% |
| **L2 硬件信创** | + 鲲鹏/飞腾 CPU | NVIDIA 仍可 | 90% |
| **L3 全栈信创** | + 昇腾/寒武纪 GPU | **重大架构调整** | 60% → 见 5.3 |

### 5.3 ⚠️ 全栈信创（L3）对工具集的冲击

**如果客户要求 L3 全栈信创（含昇腾 GPU）**：

| 工具 | NVIDIA 方案 | 昇腾替代 | 难度 |
|------|------------|---------|------|
| ms-swift | CUDA | MindFormers + MindSpore | 🔴 重大架构调整 |
| LMDeploy | CUDA | MindIE | 🔴 推理工具切换 |
| Unsloth | CUDA only | ❌ 不支持 | 🔴 完全弃用 |
| DeepSpeed | CUDA | DeepSpeed-NPU | 🟡 部分支持 |
| GLM-4-9B 权重 | HF 格式 | 需转 MindSpore 格式 | 🟡 转换脚本 |

**应对策略**：
- HR 项目启动前**必须确认客户是否要求 L3**
- 如要求 L3：**额外评估 4-6 周架构改造时间** + 成本 +30%
- 实操：**多数 HR 客户接受 L1（OS+DB 信创，保留 NVIDIA）**

### 5.4 国密算法（可选）

如客户要求国密：
- SM2 / SM3 / SM4 替换 SSL/TLS（nginx ssl_ciphers 改 sm2-with-sm3）
- 数据库连接走国密协议
- 影响：**仅网关层改动**，不影响训练 / 推理本身

### 7.2 打包脚本骨架（待启动时实现）

```bash
#!/bin/bash
# scripts/deployment/build_offline_package.sh
# 在公网环境运行，打包后通过 U 盘 / 加密硬盘传到客户内网

set -e

PKG_DIR=./offline-deploy-pkg
mkdir -p $PKG_DIR/{wheels,images,models,benchmarks,code}

# 1. 下载所有 PyPI wheel
pip wheel -r requirements.txt -w $PKG_DIR/wheels/

# 2. 拉取并保存所有 Docker image
for img in \
  argilla/argilla-quickstart:latest \
  prom/prometheus:latest \
  grafana/grafana:latest \
  ghcr.io/berriai/litellm:main-latest; do
  docker pull $img
  docker save $img -o $PKG_DIR/images/$(echo $img | tr '/:' '_').tar
done

# 3. 构建本项目自建镜像
docker build -f Dockerfile.training -t mfp-training:v1 .
docker save mfp-training:v1 -o $PKG_DIR/images/mfp-training.tar
# ... inference / backend / dataprep 同理

# 4. 下载基座模型
modelscope download --model ZhipuAI/glm-4-9b-chat-1m \
  --local_dir $PKG_DIR/models/glm-4-9b-chat-1m
modelscope download --model Qwen/Qwen2.5-14B-Instruct \
  --local_dir $PKG_DIR/models/qwen2.5-14b-instruct

# 5. 下载 benchmark 数据
opencompass --download ceval cmmlu --out $PKG_DIR/benchmarks/
git clone https://github.com/mengze-hong/QualBench $PKG_DIR/benchmarks/qualbench

# 6. 打包项目代码
git archive --format=tar HEAD | tar -x -C $PKG_DIR/code/

# 7. 生成 SHA256 校验
find $PKG_DIR -type f -exec sha256sum {} \; > $PKG_DIR/CHECKSUMS.sha256

# 8. 总包
tar -czf offline-deploy-pkg-$(date +%Y%m%d).tar.gz $PKG_DIR/
```

### 7.3 传输方式

| 方式 | 适用情景 |
|------|---------|
| 加密硬盘（推荐）| 100 GB 总包，硬盘 1 TB 富余 |
| U 盘多次拷贝 | 不推荐，容易遗漏 |
| 数据二极管 | 客户已有单向数据传输设施 |
| 内网专线 | 客户有公司间专线 |

**安全要求**：
- 离线包必须**国密 SM4 加密**（如客户要求国密）
- 传输过程**双人押运**（涉密物品标准流程）
- 客户内网解压前**校验 SHA256**
