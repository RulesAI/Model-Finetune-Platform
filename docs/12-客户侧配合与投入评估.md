# 客户侧配合与投入评估（售前/实施标准件）

> **本文定位**：回答客户/项目另一个高频问题——「微调这件事，我们（客户）侧要配合做什么？多频繁？投入多大？」
> 与 [`11-微调数据量评估方法.md`](11-微调数据量评估方法.md) 配套，构成售前/实施对客户侧投入的标准应答。
> 覆盖三类模型（文本 LLM / 多模态 VLM / 纯视觉 CV，定义见 [`10-...决策框架.md`](10-场景准入与微调路径决策框架.md) §1.1）。
>
> **核心论点**：**客户最大的误解是"我给你数据就行，模型你们做"。真相是——客户侧领域专家在项目关键路径上，
> 且贯穿全程（不只前期）。这件事谈不清，项目必卡死。售前阶段必须把客户侧工作量从隐性变量变成显性条款。**

---

## 1. 客户侧三种角色（资深专家是瓶颈）

| 角色 | 干什么 | 关键路径? |
|---|---|---|
| **领域专家（资深）** | 定标注规范、标注仲裁、抽检、建评估集/金标准、评审结果、持续学习的修正确认 | ⭐ **是，且是稀缺瓶颈** |
| **标注员（初/中级）** | 按规范做量产标注（CV 尤其重） | 是（体力侧） |
| **数据/IT 对接人** | 数据导出/脱敏/传输、内网环境/算力、业务系统（MES/HIS/ERP）API 对接、部署支持 | 是（涉密内网更重） |
| **项目/业务负责人** | 需求确认、定验收门槛、协调专家工时、决策 | 是（决定专家给不给力） |

> **核心风险**：资深专家（如 NDT-RT II 级、资深 HR、主任中医师）往往是客户最忙、最不愿抽出来的人。
> 专家不到位 = 标注规范立不住 + 评估集建不起来 + 结果没人能判 → 项目空转。
> **必须在合同/项目计划里锁定专家工时。**

## 2. 按阶段的客户配合清单 + 投入强度

（对齐本平台 5 阶段，见 [`CLAUDE.md`](../CLAUDE.md) §七；强度 ●○○=轻 / ●●○=中 / ●●●=重）

| 阶段 | 客户要配合什么 | 主要角色 | 投入强度 |
|---|---|---|---|
| **0 启动** | 提供素材清单、授权脱敏、定验收门槛、指派专家 | 业务+IT+专家 | ●●○ |
| **1 数据工程** | **定标注规范、标注、仲裁难例、抽检**；数据导出脱敏 | **专家（关键）+标注员+IT** | **●●●（最重）** |
| **2 SFT** | 抽审训练样本、反馈"对不对/像不像" | 专家 | ●●○ |
| **3 DPO/偏好** | 专家做成对判断（A/B 哪个输出更好） | 专家 | ●●○ |
| **4 评估** | **建/确认评估集与金标准、判结果是否达标** | **专家（关键）** | **●●●** |
| **5 部署** | 业务系统对接联调、内网/算力、试运行陪跑 | IT+业务 | ●●○ |
| **持续学习（上线后）** | 人工修正线上结果、资深二次确认、定期复评 | 专家+终端用户 | ●○○~●●○（**长期反复**） |

> **两头重、中间点状**：Phase 1（标注）和 Phase 4（评估）是客户投入的两座大山，都压在资深专家身上。

## 3. 三类模型的客户工作量结构差异（最实用）

| | 客户工作量长什么样 | 主力 | 工具 |
|---|---|---|---|
| **① 文本 LLM** | 量不大但**认知重**：专家亲自写/改"好答案"、判输出质量（要动领域脑子，代不了） | **资深专家直接上** | 一般文本工具 |
| **② 多模态 VLM** | 比 LLM 更重：既看图又配文，单条耗时更高 | 专家+看图人员 | 图文标注工具 |
| **③ 纯视觉 CV** | **体力最重**：逐框/逐像素标注，实例数大；需专家定"什么算缺陷"+仲裁难例+抽检 | **标注员出量 + 资深专家把关** | **必须 CVAT 等标注平台** |

> 一句话：**CV 是"人多+专家把关"（体力密集），LLM/VLM 是"专家亲自上"（认知密集）。**
> 估人力时：CV 算"标注员人月 + 专家抽检工时"；LLM/VLM 算"专家直接投入工时"。

## 4. 频率：三段，客户最易忽略第三段

1. **前期密集段**（Phase 1–2，约数周–1~2 月）：标注+规范，专家高强度、标注员满负荷。
2. **中期评审段**（Phase 3–4，点状）：专家分批做偏好判断、评估验收，每轮数天。
3. **上线后持续段**（长期，月度/季度循环）：⚠️ **客户几乎都忽略** —— 模型不是"交付即终结"：数据漂移、新案例、新标准会让效果衰减，**必须持续喂人工修正 + 专家定期复评**才能维持（反馈样本占新版训练集 ≥20%、季度迭代）。**售前不讲清这段，客户会当一锤子买卖，后期运维预算和专家工时都没留。**

## 5. 工作量怎么估 + 给客户的准确回答

与"数据量"同一哲学——**不拍死工时数，给"角色 × 阶段 × 强度"矩阵 + 数量级锚点 + 合同建议**：

**数量级锚点（按场景校准，非承诺）：**
- **资深专家**：活跃期约 **0.3–0.5 FTE**（规范+仲裁+评估评审），上线后运维降到 **~0.1 FTE**（定期复评）。
- **标注员（CV 场景）**：按"实例数 × 单实例工时"算，数千实例量级 = **人周到人月**；重标注场景常配 5–8 名标注员 + 2–3 名资深专家三级流程。
- **数据/IT**：前期数据导出脱敏 **数天–数周**（涉密内网更重）+ 部署集成 **数周** + 偶发支持。

**给客户的回答模板（可直接照念）：**
> "微调不是'你们给数据、我们闭门做'。需要你们配合三件事：**① 前期出领域专家定标注规范并做标注/仲裁（最重）；② 中期专家判结果是否达标；③ 上线后持续提供人工修正 + 专家定期复评**。其中**资深专家是关键资源**，我们建议在项目计划里锁定专家工时（活跃期约 0.3–0.5 人力、运维期约 0.1 人力）。CV 类还要配标注团队。这部分投入我们会按阶段和角色给你们一张明确的配合清单，不会临时抓人。"

价值：**把客户侧投入显性化、提前锁定**——既避免项目因专家不到位而空转，也避免客户事后觉得"怎么还老找我们"。

---

## 6. 对平台/售前的含义

本文这张"客户配合清单 + 投入强度矩阵"，应与数据量评估（docs/11）一起成为**售前标准交付件**：签约前就让客户看到自己要投入什么、投多久、投多少，把"客户侧工作量"从隐性变量变成合同条款。
