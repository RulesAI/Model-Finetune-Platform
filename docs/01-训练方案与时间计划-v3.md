## 二、关键判断（继承 v1+v2，无变化）

### 2.1 数据量决定不做 CPT

| 阶段 | 行业典型 | 现有 | 差距 |
|------|--------|------|------|
| CPT（继续预训练）| 10 亿+ token | 140 万 token | 1000× |
| SFT | 3k-10 万条 | 5k-15k 条 | 匹配 |
| DPO | 200-2000 对 | 充足 | 匹配 |

跳过 CPT，从 SFT 起步。

### 2.2 事实精度问题谁负责？

| 需求 | v2 处理 | v3 处理 |
|------|--------|--------|
| 政策条款数字精度 | 本项目 RAG 兜底 | **划给下游业务系统**（接 GraphForge 或自建 RAG）|
| 学领域语言/风格 | 本项目微调（保留）| 本项目微调（保留）|
| 学结构化推理 | 本项目微调 | 本项目微调 |

**v3 关键认知调整**：本项目的微调模型**不保证事实精度**（这是 RAG 的职责）。本项目的承诺是"会说行业语言，会按行业惯例推理"。事实细节的精确性由下游业务系统通过 RAG 实现。

这与 v2 的混淆相反，但更符合行业最佳实践（微调与 RAG 分工明确）。

## 四、5 阶段任务清单（v3）

### Phase 1 数据工程（v2 不变）

158 篇 md → 15k 条 instruction。

**多场景扩展**：每个场景在 `scenes/<scene>/instructions/rules.yaml` 配置数据合成规则，复用通用 pipeline。

### Phase 2 SFT（v2 不变，ms-swift 主）

```bash
swift sft \
  --model_id_or_path ZhipuAI/glm-4-9b-chat-1m \
  --sft_type lora \
  --dataset data/instructions/hr-salary/sft_v1.jsonl \
  --output_dir checkpoints/hr-salary/sft-v1 \
  --learning_rate 2e-4 \
  --num_train_epochs 3
```

### Phase 3 DPO（v2 不变，ms-swift 主）

```bash
swift rlhf \
  --rlhf_type dpo \
  --model_id_or_path checkpoints/hr-salary/sft-v1 \
  --dataset data/preferences/hr-salary/dpo_v1.jsonl \
  --output_dir checkpoints/hr-salary/dpo-v1
```

### ~~Phase 4 RAG~~（v3 删除）

**移除**。RAG 整体划给 GraphForge 项目。本项目不实现任何 RAG 相关代码：
- 不集成 RAGFlow
- 不集成 LightRAG
- 不集成 BGE-M3 / BGE-reranker
- 不写文档向量化代码

如果业务方坚持要"端到端含 RAG"的交付：本项目交模型 + API，GraphForge 团队交 RAG 服务，**两个项目协同交付**（不在本项目范围内合并）。

### Phase 4 评估（v3 编号，原 Phase 5）

| 指标 | 工具 | 通过线 |
|------|------|--------|
| 训练 loss | tensorboard / swanlab | 收敛 |
| 验证集 PPL | ms-swift 内置 | 比 zero-shot 降 30%+ |
| ROUGE-L（生成）| rouge-chinese | ≥ 0.45 |
| Exact Match（抽取）| 自跑脚本 | ≥ 0.75 |
| **C-Eval / CMMLU** 通用能力回归 | OpenCompass | 比 base 掉点 < 3% |
| **QualBench HR 资格类** | OpenCompass | ≥ 80% |
| 业务测试集 | DeepEval | 人工评 ≥ 4.0/5 |

**v3 移除**：~~RAGAS 四件套~~（RAG 评估归 GraphForge）。

### Phase 5 部署（v3 编号，原 Phase 6）

| 维度 | 主选（涉密单机）| 备选（dev/SaaS）|
|------|-------------|---------------|
| 引擎 | **LMDeploy** | vLLM |
| 量化 | INT4 / FP8 | INT8 |
| 端口 | 8016 | 8016 |
| 协议 | OpenAI 兼容 | OpenAI 兼容 |

```bash
lmdeploy serve api_server \
  --model-name glm-4-9b-hr-salary-v1 \
  --model-path checkpoints/hr-salary/dpo-v1 \
  --quant-policy 4 \
  --server-port 8016
```
