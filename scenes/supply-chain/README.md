# Scene: Supply-Chain（供应链）

> **状态**：📋 规划中（占位）
> **预计启动**：2026-Q3 ~ Q4（待客户合作落定）
> **占位日期**：2026-05-25

---

## 一、场景定位（规划）

为**供应链密集型企业**（零售 / 电商 / 物流 / 制造）的业务流程理解、异常预警**提供 AI 助手的基础模型能力**。

### 注意：本场景只产出模型 + API（v3 严格定位）

| 本场景产出 | 由下游业务系统产出 |
|----------|------------------|
| 微调后的模型 | 业务系统对话 UI |
| OpenAI 兼容 API | ERP / WMS / TMS 实时查询（Tool use）|
| System Prompt 模板 | 业务流程图谱 + RAG（GraphForge）|
| 评估报告 | 异常处理工单流程 |

### 潜在下游消费方

- **Supply Nexus Agent**（供应链专家知识助手）
- **EvoChain-Agent**（多 Agent 编排消费 LLM）
- **GraphForge** + SupplyGraph（图谱 + RAG 服务）

## 三、场景特征（预测）

| 维度 | 预测取值 | 工具选型影响 |
|------|---------|------------|
| 多模态需求 | 低（结构化 + 文本）| 文本模型即可 |
| 涉密等级 | 中-高（供应商价格 / 客户订单）| 数据本地化 |
| 实时性要求 | 高（业务咨询要秒级）| LMDeploy 低延迟优化 |

### 与 HR、Manufacturing 场景的差异

1. **强结构化数据**：大量 xlsx / json 需先转 markdown 再训练
2. **Tool use 需求由下游解决**：本项目不实现 ERP / WMS API 编排，下游 Agent 系统做
3. **业务方学习曲线短**：供应链高频问题相对固定，可能不需要复杂 DPO

## 八、协同关系（潜在最高复用价值）

- **SupplyGraph / GraphForge**：图谱建模直接复用，本场景模型作 RAG 后端
- **Supply Nexus Agent**：本场景模型作 LLM 后端
- **EvoChain-Agent**：Tool use 编排消费本场景模型
- **manufacturing 场景**：制造 ↔ 供应链业务自然连接

**最大复用价值**：团队既有项目几乎全可作为下游消费方，启动后预期 2 周可交付（首发场景 5 周的 40%）。
