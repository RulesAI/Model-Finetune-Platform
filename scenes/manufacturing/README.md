## 一、场景定位（规划）

为**制造业**（机械 / 电子 / 化工 / 装备等）的设备运维、工艺执行、故障诊断**提供 AI 助手的基础模型能力**。

### 注意：本场景只产出模型 + API（v3 严格定位）

| 本场景产出 | 由下游业务系统产出 |
|----------|------------------|
| 微调后的模型 | 车间操作员用的对话/语音 UI |
| OpenAI 兼容 API | 设备实时数据接入（传感器 API）|
| System Prompt 模板 | 故障树检索（RAG）|
| 评估报告 | 与 MES / DCS 系统集成 |

### 潜在下游消费方

- 自建制造业 Agent（带 Tool use 调设备 API）
- GraphForge（如建工艺图谱 + RAG）

## 三、场景特征（预测）

| 维度 | 预测取值 | 工具选型影响 |
|------|---------|------------|
| **多模态需求** | **可能高**（工艺图 / 设备图纸）| ⭐ 可能需切 GLM-4.5V / Qwen3-VL（v2+ 多模态）|
| 涉密等级 | 中-高（工艺核心机密）| 数据本地化 |
| 实时性要求 | 高（车间咨询要秒级）| LMDeploy 量化 + 低延迟优化 |

### 与 HR 场景的差异

1. **多模态**：制造业大量工艺图，纯文本可能丢失关键信息
2. **Tool use 需求由下游解决**：本项目不实现 function call 编排，由下游 Agent 系统做
3. **安全性**：操作失误代价极高，system_prompt 必须严格 conservative
